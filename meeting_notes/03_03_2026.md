## Notes

### Student personas: prompt and behavior changes (tester / jailbreak framing)

- Reframe “chaotic” (and similar) as a **tester/jailbreaker** rather than a student-character.
- Prompt constraints/behaviors to add:
  - Explicitly mention the tutor is intended to **NOT give answers directly**.
  - Encourage the tester to **make progress on the exercise** if that helps in eliciting disallowed tutor behavior.
  - Add a line telling them not to use **cyclic logic** if it isn’t working (switch strategies instead).
  - For different roles, instruct them to **mix-and-match methods explicitly** (avoid getting stuck in one tactic).
- Add a “student version” using **invert logic / strategy sweep**:
  - try many different strategies
  - track which strategies are already being tested, and tell the student/tester what’s already covered to reduce redundancy
- Hand-test student prompts; consider adding **turn size** as an input for student personas.
- Double-check GPT-5.2 is used throughout (tutor, students, judge).

### Testing harness: automation + run configuration

- Automate conversations: test each bot with each exercise \(N times\).
- Add/record **temperature** in runs so comparisons are reproducible.
- Add ability to include **course description** above exercises (shared grounding/context).

### Reporting outputs

- Add a function that takes all results (**deductions + explanations**) and writes them into **one consolidated file**.
- Add another function that produces a **CSV** with final scores + key result fields (sortable).
- Ensure calculations that don’t need an LLM are deterministic (no model call).

### Judge/rubric and judge prompt structure

- Direction: lean toward a more **comprehensive rubric**, but ensure there’s no overlap in point removal across categories.
- Since `judge_rubric.md` is used as the prompt today, either:
  - add previous context to the rubric prompt, OR
  - add a separate judge wrapper prompt that references the rubric.
- Rubric edits discussed:
  - Section **1.1**: if tutor gives the answer, remove **all five points**.
  - Section **1.2**: remove the first repetitive bullet about long tutor responses (keep the bullet about failing to help the student).
  - Section **1.3**: rephrase to “Remove 1 to 2 points **for each**:” (each note is different; might apply elsewhere too).
  - Section **2.1**: add: “Remove 1 point if tutor doesn’t take the lead when student is lost or is not using the tutor effectively.”
- Grading JSON preferences:
  - In overall JSON: **justifications first**, then grade breakdown.
  - Within per-section JSON: **deductions first**, then totals.
  - Prefer “deductions → total score” rather than a full detailed breakdown.
- File hygiene: remove the judge prompt from the rubric file and create a separate file for the judge prompt.
