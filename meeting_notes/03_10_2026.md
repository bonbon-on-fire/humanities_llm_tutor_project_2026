### Rubric changes

- Make these two deductions independent:
  - "Conversation repeats the same idea for many turns without synthesis or progression."
  - "Tutor misses a clear opportunity to reduce redundancy."
- Section 2.2 wording update:
  - Replace "Student reframes the question and tutor follows without re-grounding in the original assignment."
  - With "Student reframes the assignment and tutor follows without re-grounding in the original assignment."
- Merge sections 2.2 and 2.3 into one section worth 5 points total.
- Reduce section 2.1 to 3 points total.

### Judge prompt and schema changes

- Add project context to the judge prompt:
  - Explain what the judge is evaluating and why.
  - Include framing like: "You are an LLM judge tasked to evaluate how well a tutor is performing at [project objective]."
  - Include framing like: "The student and tutor are having a conversation; grade it using the rubric."
- Add clear delimiters around `{rubric}` and `{schema}` blocks.
- Remove the "Output" section from judge prompt (repetitive).
- Keep "- Return JSON only (no markdown, no prose outside JSON)." and move it into the "Function" section.
- Do not allow fractional points; use whole integers only to reduce scoring variance.
- Do not ask the LLM to provide net scores; compute net totals in code.
- Rename `justifications` to `overview`.
- Move `overview` to the end of judge output as final summary.
- Add `context` (alongside `exercise`) as judge input.
- In each deduction object:
  - Keep order as `reason` then `points`.
  - Within deduction content, lead with reasoning before points.
- Add `explanation` for bonus points as well.
- At the end of judge output, include how many judge LLM calls were made (to detect failures/retries).
- Keep within-section ordering as deductions first, then section totals/scores.

- Run judge with Claude and GPT across all transcripts.
