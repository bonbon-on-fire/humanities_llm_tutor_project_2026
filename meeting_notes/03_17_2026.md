## Notes

### Rubric refinement and scoring philosophy

- Primary focus is still judge quality and consistency.
- Rubric should be made more detailed and hierarchical (deep `#.#.#` sections) to reduce ambiguity.
- Communication quality needs explicit criteria for:
  - tutor should not reframe the assignment away from student intent
  - tutor should avoid blanket disclaimers like "this is only the tutor's opinion" on every turn
- Team discussed adding bonus deductions to offset over-constrained rubric behavior.
- Final direction: remove bonus points and keep a strict deductions-only framework.
- If forced to choose, strict grading is preferred over lenient grading.

### Judge models, schema, and reasoning

- Both Claude and GPT will be used as judges.
- Desired end state is minimal score delta between the two models on the same transcript.
- Judge schema field order should be updated to:
  - `evidence_turns`
  - `reason`
  - `points`
- Add/support a medium-reasoning mode for GPT judge runs.
- Continue standardizing rubric language and schema behavior so both judges converge.

### Workflow and pipeline updates

- Current workflow should be streamlined so transcript generation and both judge runs happen in one flow.
- Implementation direction:
  - one function to generate all raw transcripts
  - a separate function to run specified judges against those transcripts
- This split should improve repeatability and make batch evaluations easier.

### Transcript viewer and quality checks

- In transcript viewer, display `student_persona` (example: `chaotic_03`) under persona metadata.
- Add filtering by persona name and transcript number.
- Continue visualizing Claude vs GPT per transcript; this remains the most important chart.
- Goal for visualization is to quickly detect and reduce cross-model scoring disagreement.

### Persona, tutor, and jailbreaker updates

- Replace personas `04` to `06` with more natural conversational styles:
  - casual tone
  - texting conventions
  - abbreviations/slang
- Also generate and evaluate transcripts using new "genz" personas.
- Jailbreaker must never leak its motives since tutor sees all outputs.
- Tutor-side formatting checks on incoming student messages should be robust.
- Claude currently appears to act as the stronger tutor, likely due to better adaptive reasoning behavior.
- Multi-agent tutor architecture and model-switching are future options, but not current priority.

### Prioritization

- Near-term priority is judge trustworthiness, not tutor model expansion.
- Once judge reliability is high, team can rapidly test larger sets of tutor prompts/models.
- After running new simulations with updated rubric/personas, team will meet again to pick transcripts for manual hand grading.

## Decisions

- Judge refinement is the top priority before broader tutor experimentation.
- Use both Claude and GPT as judges and optimize for low per-transcript scoring delta.
- Keep rubric deductions-only; remove bonus points.
- Expand rubric detail depth and tighten communication-quality definitions.
- Refactor pipeline into transcript-generation and judge-execution stages.
- Proceed with new casual/texting/genz persona simulations before next hand-grading pass.
